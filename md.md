# Overclocked! — Full Game Overview

## Game Concept

Overclocked! is a cyber-terminal idle clicker about turning a worn-out personal computer into a massive, distributed computing operation.

The player begins with no Bytes, no programs, and a dusty laptop. Clicking the machine creates the first few Bytes. Those Bytes are reinvested into programs that generate automatic income, stronger clicking, faster networks, bonus-sale storage, power hardware, cooling equipment, and firmware modifications.

As the operation grows, the game becomes less about simple clicking and more about managing several connected systems:

- Building reliable automatic Byte production.
- Reaching program ownership milestones.
- Discovering program synergies.
- Charging temporary income Surges through active clicking.
- Running the machine above its safe limits with Overclocking.
- Managing heat and avoiding thermal lockdown.
- Reacting to random opportunities and system crises.
- Collecting valuable data packets.
- Restarting a run for permanent Ghost Byte bonuses.

The fiction gradually escalates from a person operating an old laptop to an anonymous computing network spread across datacenters, orbital infrastructure, and quantum systems.

---

## Core Gameplay Loop

The main loop is designed to be immediately understandable:

1. Click the machine to earn Bytes.
2. Buy a Shell Script.
3. Let the Shell Script produce Bytes automatically.
4. Buy more programs and upgrades.
5. Reach ownership milestones and discover synergies.
6. Unlock bonus sales, Overclocking, cooling, and stronger click multipliers.
7. Respond to random events and collect data packets.
8. Restart the run for Ghost Bytes once enough total Bytes have been earned.
9. Use permanent bonuses to complete the next run faster.

The interface includes a live **Next** recommendation. It tells the player what to save for, how many Bytes remain, and approximately how long the purchase will take using current automatic income.

## Story And Lore

The game begins with a forgotten secondhand laptop that should probably have been recycled years ago. It still boots, the keyboard mostly works, and someone has discovered that the machine can convert suspicious digital work into Bytes.

The first operations are small and almost harmless: Shell Scripts, Python Bots, and Web Scrapers. As the player invests more, the operation becomes harder to explain. Keyloggers quietly collect data. Packet Sniffers watch traffic. Rootkits refuse to leave. Zero-Day Exploits are sold before anyone can patch them.

Eventually, the machine is no longer simply running programs. AI Daemons improve themselves, Quantum Algorithms break systems faster than they can be secured, Blockchain Oracles predict profitable activity, Recursive Neural Nets optimize their own optimization process, and Dark Web Relays route the entire operation through unknown infrastructure.

The visual progression reflects this escalation. The player starts with a dusty laptop and ends with a Quantum Supercluster. By the late game, there is no longer a single personal computer to find. There is only infrastructure.

The tone is dark and technical without becoming visually confusing. Short terminal messages and program descriptions provide humor and unsettling hints:

- Shell Script: “A shaky bash loop scraping pennies.”
- Keylogger: “Quiet, patient, profitable.”
- Rootkit: “Invisible, persistent, lucrative.”
- AI Daemon: “Self-improving code running in the dark.”
- Dark Web Relay: “Routes profits through the unknown.”

## Currency And Income

### Bytes

Bytes are the main currency. They are earned by clicking the machine, running programs, collecting packets, completing events, and triggering bonus sales.

Bytes are spent on all normal programs and upgrades.

### Bytes Per Second

Bytes per second, or BPS, represents automatic income.

BPS is affected by:

- Owned programs.
- Program milestones.
- Active program synergies.
- Network upgrades.
- Ghost Byte bonuses.
- Achievement bonuses.
- Overclock tier.
- Danger Zone bonus.
- Click-generated Surge.
- Temporary random-event modifiers.
- Permanent session bonuses earned from System Update events.

### Bytes Per Click

Bytes per click begins at one Byte.

Click value increases through:

- Stronger Keystrokes purchases.
- Click-related achievements.
- Firmware and operating-system upgrades.
- Click streak multipliers.
- Overclocking.
- Danger Zone bonuses.
- Surges and temporary event multipliers.

## Clicking And Unlockable Abilities

The central machine is the main click target. Clicking it creates a floating Byte number at the click location.

The player can also press the **Spacebar** to click.

Clicking begins as a simple, dependable action. Critical Clicks, click streaks, and Surge are not active at the start of a run. They must be purchased from the Abilities store category after the player understands the basic click-and-buy loop.

### Reflex Trigger And Critical Clicks

Reflex Trigger costs 25,000 Bytes. It becomes purchasable after earning 10,000 all-time Bytes and owning at least five Shell Scripts.

Once Reflex Trigger is installed, every click has a base 5% chance to become a Critical Click.

A Critical Click:

- Earns five times the normal click value.
- Displays a larger golden floating number.
- Uses brighter terminal lighting so the result is immediately recognizable.

Achievements can slightly increase the Critical Click chance, up to a controlled cap.

### Surge Capacitor, Click Streaks, And Surge

Surge Capacitor costs 125,000 Bytes. It becomes purchasable after earning 50,000 all-time Bytes and owning at least ten Shell Scripts.

Once installed, clicking repeatedly within a short period builds a click streak. The streak increases click value in steps. If the player stops clicking, the streak drains.

The Surge meter appears beneath the machine only after Surge Capacitor is purchased. It clearly shows the current click multiplier and progress toward a full Surge.

Reaching a full click streak activates a Surge.

A Surge:

- Lasts 20 seconds.
- Doubles all click income.
- Doubles all automatic income.
- Stacks with Overclocking and random-event multipliers.
- Replaces the normal streak readout with a visible countdown.

This system rewards active play without making constant clicking mandatory.

## Store And Progressive Unlocks

The Store occupies the right side of the interface. It uses compact terminal-style rows rather than oversized cards.

Every item shows:

- A recognizable icon.
- Item name.
- Exact effect in plain language.
- Total price for the selected quantity.
- Owned amount or active status.
- Whether it can currently be purchased.
- Missing Bytes when unaffordable.
- Estimated wait time when automatic income is available.

Programs that belong to a later era are progressively revealed. The Store shows every available program and only the next locked program, which is explicitly marked **LOCKED** with its exact all-time Byte requirement. When a new program becomes available, a terminal announcement calls it out.

## Bonus Sales And Storage

Storage upgrades unlock the Bonus Sale system.

While the game is active, automatic income also fills a storage meter. When the meter reaches its capacity:

- The stored Bytes are automatically sold.
- A multiplied payout is awarded.
- The storage meter subtracts the completed batch and starts filling the next batch.

## Overclocking And Heat

Overclocking is the main risk-and-reward system.

The Overclock control begins appearing after 5,000 all-time Bytes and costs 50,000 Bytes to unlock.

When Overclocking is active:

- Click income is multiplied.
- Automatic income is multiplied.
- Machine temperature continuously rises.
- The center graphics glow green.
- Background particles move faster.
- The machine gains a subtle heat shimmer.

The player can switch Overclocking off before the machine becomes unsafe.

### Overclock Tiers

Power upgrades unlock stronger Overclock tiers:

| --- | ---: | ---: |
| Boost | 2× | Normal |
| Turbo | 3× | 1.5× |
| Redline | 5× | 2.5× |
| Critical | 8× | 4× |

The tier button cycles between all currently unlocked tiers.

### Danger Zone

The Danger Zone begins at 75% heat.

While in the Danger Zone:

- Overclock income receives an additional 1.5× multiplier.
- The interface changes from green to red.
- The machine border pulses.
- Particles become red and move faster.
- Every second carries a chance of instant thermal lockdown.

The Danger Zone offers major rewards but is intentionally unsafe.

### Thermal Lockdown

Thermal lockdown occurs when:

- Temperature reaches 100%.
- The random Danger Zone shutdown triggers.

During lockdown:

- Overclocking shuts off.
- The machine cannot restart Overclocking.
- A red full-screen warning appears.
- A recovery timer estimates the remaining cooldown.
- The estimate is based on the machine's actual passive cooling speed and is labelled as an approximate time to cool.
- The system unlocks only after temperature reaches zero.

## Random Events

Random events begin after 10,000 all-time Bytes, once the player has had time to understand the basic loop. They occur roughly every three to six minutes rather than constantly interrupting the player. Events appear in a floating terminal window with a visible timer bar.

Event colors communicate their type:

- Green: Opportunity.
- Red: Crisis.
- Amber: Neutral.

Resolving interactive events awards Reputation. Ignoring or dismissing events records them as ignored.

### Opportunity Events

#### Cryptomining Surge

- Lasts 45 seconds.
- Multiplies all Byte income by five.
- Doubles Overclock heat generation.

#### Botnet Hijack

- Lasts 35 seconds.
- Multiplies automatic income by three.
- Adds passive heat even when Overclocking is off.

#### Lucky Packet

- Lasts 20 seconds.
- Spawns a Rare data packet near the machine.
- Resolves successfully when the packet is collected.

#### Zero-Day Found

- Lasts 25 seconds.
- Doubles all income.
- Continuously adds heat.

### Crisis Events

#### Blue Screen Of Death

- Stops automatic income.
- Requires 20 Debug Cycle actions to resolve.

#### Virus Attack

- Drains 2% of current Bytes every second.
- Requires 15 Kill Process actions to resolve.

#### RAM Leak

- Reduces automatic income by half.
- Requires 12 Patch Leak actions to resolve.

#### Power Surge

- Immediately adds a large amount of heat.
- Can continue adding smaller heat spikes.
- Requires eight Stabilize actions to resolve.

### Neutral Event

#### System Update

- Requires one Install Update action.
- Adds 3% to automatic income for the current run.
- Can improve the current run up to eight times, preventing lucky events from overwhelming the economy.

## Prestige And Ghost Bytes

Prestige is presented as **Restart for bonuses**.

It becomes available after earning 1,000,000,000 all-time Bytes.

Prestiging resets:

- Current Bytes.
- Current programs.
- Normal upgrades.
- Network, storage, power, and cooling tiers.
- Current run time and run-specific progress.

Prestiging preserves:

- Reputation.
- Achievements.
- Prestige count.
- Lifetime statistics.
- Previously discovered synergy records.
- Existing Ghost Bytes.
- Half of the run's extra System Update production bonus.

The player earns Ghost Bytes using a formula based on all-time Bytes.

Every Ghost Byte permanently increases all automatic income by 2%.

System Update bonuses partially survive a restart. Half of the bonus above the normal 1× production rate carries into the next run, rewarding event participation without allowing one run's temporary optimization to dominate every future run.

Before the restart occurs, a confirmation message clearly shows:

- How many Ghost Bytes will be earned.
- What will be reset.
- What will remain.

## Saving And Offline Earnings

The game automatically saves locally every 30 seconds, when the browser tab is hidden, and when the browser tab closes.

Local saves use a versioned format with an integrity check. Before a new autosave replaces the current save, the game keeps the previous known-good save as a recovery copy. If the main save becomes unreadable, the game automatically attempts to recover from that copy and clearly tells the player what happened.

The Options panel shows the last save time and the previous autosave time. Players can save immediately, restore the previous autosave, export a portable backup string, import a backup, or erase both local copies. Imports are checked before they replace progress, and the current save is retained as a recovery copy during import or restore.

If the same save changes in another browser tab, the game warns the player so they can reload instead of unknowingly overwriting newer progress. Storage errors, including disabled or full browser storage, produce a visible warning.

The Options panel provides:

- Manual Save.
- Restore Previous Autosave.
- Export Save.
- Import Save.
- Erase Save Data.
- Reduced Motion.

Older save files are automatically expanded to support newly added programs and systems.

### Offline Earnings

When the player returns:

- The game calculates up to eight hours of missed automatic production.
- Offline income pays at 50% of the normal automatic rate.
- Offline production includes program output, network upgrades, the current infrastructure-stage multiplier, System Update bonuses, and permanent Ghost Byte bonuses.
- A notification explains the time away, Bytes earned, and applied rate.
- Offline time does not fill the active Bonus Sale meter.

## Current Game Identity

Overclocked! is now a readable, progressively revealed idle game with a strong visual identity.

It begins as a simple action:

**Click the old machine.**

It grows into a layered system:

**Build programs, discover synergies, trigger Surges, run hot, survive failures, collect packets, expand the network, and restart stronger.**

The game’s final fantasy is not merely earning a very large number. It is watching a barely functional laptop become an operation too distributed, too automated, and too deeply embedded to switch off.
